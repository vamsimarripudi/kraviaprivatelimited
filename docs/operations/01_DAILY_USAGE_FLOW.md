# Daily Usage Flow

## Purpose

Use this flow to keep KRAVIA Company OS current without creating unnecessary records.

## Who Should Use This

- Founders
- Directors
- Team members with assigned operational work

## Daily Steps

1. Log in.
2. Review notifications.
3. Review announcements.
4. Check open tasks.
5. Check overdue tasks.
6. Check upcoming compliance items.
7. Review document requests or uploads.
8. Update records that changed since the last login.
9. Add notes only when they help future review.
10. Log out when work is complete.

## What To Update Daily

Update only real changes:

- Task status
- Compliance status
- Meeting action item status
- Contact follow-up date
- Product milestone or risk
- New document upload
- Important announcement

## What Not To Do Daily

Do not:

- Add placeholder records.
- Duplicate existing documents.
- Change financial records without source data.
- Archive records without review.
- Use notes for informal chat.

## End Of Day Check

Before logging out:

1. Confirm your updates were saved.
2. Confirm important changes appear in audit logs where expected.
3. Confirm any urgent task has an owner and due date.
4. Confirm no private files were left on a shared device.
