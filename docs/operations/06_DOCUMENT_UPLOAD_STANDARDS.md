# Document Upload Standards

## Purpose

Use these standards to keep KRAVIA company documents organized, searchable, and safe.

## Before Uploading

1. Confirm the document belongs in KRAVIA Company OS.
2. Search Documents to avoid duplicates.
3. Confirm the file is final or clearly identified as a draft.
4. Confirm the file does not contain unrelated personal information.
5. Confirm you have permission to upload it.

## Required Metadata

Every upload should include:

- Clear title
- Correct category
- Short description, if useful
- Correct status
- Correct version when replacing or updating

## Naming Standard

Use clear names that include:

- Company or document subject
- Document type
- Date if relevant
- Version if relevant

Avoid:

- `final-final`
- `new`
- `scan`
- `document`
- Personal names unless required
- Unclear abbreviations

## Category Selection

Choose the closest category:

- Incorporation Certificate
- MOA
- AOA
- Company PAN
- TAN
- GST
- Board Resolution
- Rental Agreement
- Bank Document
- Trademark Document
- Startup India Document
- Agreement
- Other

Use Other only when no category fits.

## Versioning

When updating a document:

1. Keep the old record when history matters.
2. Upload the new file as a new version if version tracking is available.
3. Update title or description only when needed.
4. Do not overwrite records to hide history.

## Sensitive Documents

Handle these carefully:

- PAN
- TAN
- Bank documents
- Agreements
- Board resolutions
- Investor documents
- Legal notices
- Identity documents

Download sensitive documents only when required.

## Upload Rules

- Do not upload fake documents.
- Do not upload temporary screenshots as official records.
- Do not upload password lists.
- Do not upload unrelated personal files.
- Do not share local storage paths.
- Use the protected download action inside the system.
